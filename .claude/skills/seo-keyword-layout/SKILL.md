# SEO关键词布局策略

## 概述

这是一个交互式工作流程,引导用户完成工具站的SEO关键词布局。通过4个步骤(关键词分层 → 模块布局 → 密度优化 → AI适配),为工具站制定系统化的关键词布局方案。

## 执行流程

当此 skill 被触发时,立即启动以下交互式流程:

### 第0步: 了解项目背景

首先使用 AskUserQuestion 收集项目基本信息:

**必须询问的问题:**
1. 工具站的核心功能是什么?(例如: PDF转换、图片压缩、代码格式化等)
2. 目标用户群体是谁?(例如: 学生、设计师、开发者、职场人等)
3. 当前是新站上线还是现有站点优化?
4. 是否已有关键词列表?如果有,请提供

**收集方式:**
- 如果用户已在消息中提供部分信息,直接使用,只询问缺失的部分
- 如果用户完全没提供,一次性询问所有问题
- 使用选择题+文本输入的方式,提高效率

### 第1步: 关键词分层归类

根据用户提供的信息,执行以下操作:

**如果用户已有关键词列表:**
1. 读取用户提供的关键词列表
2. 自动将关键词分为4类: 核心词、长尾精准词、场景词、问题词
3. 展示分类结果,询问用户是否需要调整

**如果用户没有关键词列表:**
1. 根据工具功能和目标用户,生成建议的关键词列表
2. 按4类展示: 核心词、长尾精准词、场景词、问题词
3. 每类提供3-5个示例
4. 询问用户是否需要补充或修改

**分类标准:**
- **核心词**: 工具站最核心的业务词(搜索量大、竞争高)
- **长尾精准词**: 带场景/需求的细分词(转化意图强)
- **场景词**: 绑定用户身份/使用场景的词(针对特定人群)
- **问题词**: 用户搜索的疑问类词(长尾流量)

### 第2步: 按模块规划布局

基于分层后的关键词,为网站各模块规划具体布局:

**必须输出的布局方案:**

1. **首页布局方案**
   - Title标签建议(核心词前置,60字符内)
   - Meta Description建议(核心词+行动引导,160字符内)
   - Hero区标题建议(核心词+用户痛点)
   - 功能导航栏建议(每个功能用长尾词命名)

2. **功能详情页布局方案**
   - 为每个主要功能页提供Title建议
   - 正文开头段落建议(100字内,嵌入核心词+痛点)
   - FAQ问题建议(3-5个问题词)

3. **博客内容页布局方案**
   - 建议3-5个博客文章标题(长尾词+数字/疑问)
   - 每篇文章的H2小标题建议
   - 内链锚文本建议

4. **全站通用模块建议**
   - 面包屑导航示例
   - 404页面文案建议
   - 搜索框提示语建议

**输出格式:**
使用清晰的表格或列表,让用户可以直接复制使用

### 第3步: 密度优化建议

为用户提供关键词密度优化的具体建议:

**必须包含:**
1. 核心词密度控制建议(2%-5%)
2. 同义词替代方案(为核心词提供3-5个同义词)
3. 自然度检查清单(5-7条具体检查项)

**如果用户提供了现有页面内容:**
1. 分析当前关键词密度
2. 指出密度过高或过低的地方
3. 提供具体的修改建议

### 第4步: AI搜索适配方案

为用户提供AI搜索优化的具体实施方案:

**必须输出:**
1. **Schema标记代码**
   - 为工具页生成SoftwareApplication类型的JSON-LD代码
   - 嵌入核心词、功能描述、使用方式
   - 提供完整可用的代码片段

2. **结构化内容建议**
   - H2/H3标签结构建议
   - 推荐的小标题列表("核心功能"、"使用教程"等)

3. **AI检索摘要模板**
   - 为首页生成100-150字的摘要
   - 包含核心词+核心优势
   - 提供2-3个版本供选择

**输出方式:**

## 关键词分类参考

执行流程中需要用到的关键词分类标准:

**核心词**
- 工具站最核心的业务词
- 特征: 搜索量大、竞争高
- 示例: "在线PDF转换器"、"AI图片压缩工具"
- 布局位置: 首页、权重最高的模块

**长尾精准词**
- 带场景/需求的细分词
- 特征: 转化意图强、竞争适中
- 示例: "免费PDF转Word无水印"、"学生用PDF转Word工具"
- 布局位置: 功能页、博客页

**场景词**
- 绑定用户身份/使用场景的词
- 特征: 针对特定人群、场景明确
- 示例: "设计师用的图片压缩工具"、"职场人PDF批量处理"
- 布局位置: 功能介绍、博客内容

**问题词**
- 用户搜索的疑问类词
- 特征: 以问题形式出现、长尾流量
- 示例: "PDF转Word怎么不丢格式"、"图片压缩后模糊怎么办"
- 布局位置: FAQ模块、博客问答内容

## 模块布局规则参考

执行流程中需要遵循的布局规则:

**首页布局(权重最高)**
- Title标签: 核心词前置+优势词,控制60字符内
- Meta Description: 核心词+行动引导,控制160字符内
- Hero区标题: 核心词+用户痛点
- 功能导航栏: 每个功能用长尾词命名
- 底部版权区: 重复1次核心词

**功能详情页布局(长尾词+场景词)**
- 页面Title: 长尾词+场景
- 正文开头: 100字内嵌入核心词+痛点
- 使用步骤: 每步自然嵌入长尾词
- FAQ模块: 问题词作标题,回答中嵌入核心词
- 操作按钮: 核心词+引导语
- 图片Alt文本: 添加含核心词的描述

**博客内容页布局(问题词+场景词)**
- 文章标题: 长尾词+数字/疑问
- H2/H3小标题: 场景词或问题词
- 正文布局: 开头结尾各嵌入1次核心词,中间自然融入长尾词
- 内链锚文本: 用长尾词链接到功能页
- 文章标签: 添加3-5个相关关键词

**全站通用模块(补充布局)**
- 面包屑导航: 嵌入核心词/长尾词
- 404页面: 核心词+引导
- 搜索框提示语: 核心词/长尾词引导

## 密度优化标准

**控制密度范围**
- 核心词密度: 2%-5%
- 长尾词密度: 无需刻意控制,自然融入
- 避免单句重复关键词

**使用同义词替代**
- 用同义词增加内容自然度
- 示例: "PDF处理工具"替代"PDF转换器"、"格式还原"替代"不丢格式"

**人工校验**
- 布局完成后通读页面
- 确保语句通顺、有价值
- 优先满足用户需求,而非SEO指标

## AI搜索适配标准

**添加Schema结构化数据**
- 为工具页添加SoftwareApplication类型Schema
- 嵌入核心词、功能、使用方式
- 让AI搜索引擎快速识别工具价值

**结构化内容块**
- 用H2/H3标签拆分功能介绍、使用步骤
- 使用"核心功能"、"使用教程"、"用户评价"等小标题
- 方便AI抓取并生成精准回答

**AI检索摘要**
- 在页面开头设置适配AI搜索的摘要
- 包含核心词+核心优势
- 控制在100-150字内

## 关键原则

1. **交互式引导**: 主动询问用户信息,不要等待用户提供
2. **分步执行**: 严格按照第0-5步的顺序执行,每步完成后再进入下一步
3. **具体可用**: 输出的所有内容必须是具体可用的,不是泛泛而谈
4. **保存文档**: 最后必须询问用户是否保存完整方案为文档
5. **持续优化**: 根据用户反馈随时调整方案
