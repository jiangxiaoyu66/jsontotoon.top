# 网站开发前 SEO 调研工作流程指南

---

## 📋 **完整工作流程概览**

```
阶段1: 产品与用户定位
   ↓
阶段2: 关键词研究
   ↓
阶段3: 竞品分析
   ↓
阶段4: SEO策略制定
   ↓
阶段5: 外链渠道规划
   ↓
阶段6: 内容规划
   ↓
阶段7: 技术SEO配置
   ↓
阶段8: 执行计划与KPI
```

---

## 🎯 **阶段1: 产品与用户定位**

### 目标
明确产品是什么、解决什么问题、服务谁

### 具体工作

**1.1 产品定义**
- [ ] 产品核心功能是什么？
- [ ] 产品解决什么痛点？
- [ ] 产品的独特价值是什么？
- [ ] 产品的技术特点是什么？

**1.2 用户画像**
- [ ] 目标用户是谁？（职业、角色）
- [ ] 用户的使用场景是什么？
- [ ] 用户的技术水平如何？
- [ ] 用户的痛点是什么？

**1.3 搜索意图分析**
- [ ] 用户会搜索什么词找到你？
- [ ] 用户搜索的目的是什么？
  - 学习了解（信息型）
  - 使用工具（交易型）
  - 对比选择（对比型）
  - 解决问题（问题型）

**1.4 市场定位**
- [ ] 你的产品在市场中的位置？
- [ ] 与现有解决方案的差异？
- [ ] 目标细分市场是什么？

### 输出物
- ✅ 产品定位文档
- ✅ 用户画像（3-5个典型角色）
- ✅ 用户搜索意图地图

## 🔍 **阶段2: 关键词研究**

### 目标
找到所有相关关键词，确定核心关键词和长尾词策略

### 具体工作

**2.1 关键词收集**

**方法1: 头脑风暴**
- [ ] 列出产品相关的所有词汇
- [ ] 列出用户可能搜索的问题
- [ ] 列出竞品名称和相关词

**方法2: Google 搜索建议**
- [ ] Google 搜索框自动补全
- [ ] Google 搜索结果页底部"相关搜索"
- [ ] Google "People Also Ask" 问题

**方法3: 专业工具**
- [ ] Semrush 关键词魔法工具
- [ ] Ahrefs 关键词浏览器

**方法4: 竞品关键词**
- [ ] Semrush 竞品关键词分析
- [ ] Similarweb 流量来源关键词
- [ ] Ahrefs 竞品排名关键词

**方法5: 社区挖掘**
- [ ] Reddit 相关讨论
- [ ] Stack Overflow 问题
- [ ] Quora 热门问题
- [ ] GitHub Issues
- [ ] Twitter/X 话题

**2.2 关键词分类**

按类型分类：
- **核心词**（Head Terms）：1-2个词，搜索量大，竞争高
- **次级词**（Body Terms）：2-3个词，搜索量中等
- **长尾词**（Long-tail）：3+个词，搜索量小，转化高

按意图分类：
- **信息型**（Informational）："what is", "how to", "guide"
- **交易型**（Transactional）："buy", "download", "free", "online"
- **导航型**（Navigational）：品牌名、产品名
- **对比型**（Comparison）："vs", "alternative", "best"

**2.3 关键词评估**

收集以下数据：
- [ ] 月搜索量（Search Volume）
- [ ] 关键词难度（Keyword Difficulty, KD）
- [ ] CPC（每次点击成本）
- [ ] 搜索趋势（Google Trends）
- [ ] SERP 特征（Featured Snippet, Video等）

**2.4 关键词对比与筛选**

使用工具对比：
- [ ] **Google Trends**: 对比多个关键词的趋势和地域分布
- [ ] **Semrush**: 对比搜索量和难度
- [ ] **Similarweb**: 对比流量潜力

筛选标准：
- ✅ 搜索量 > 最低阈值（如100/月）
- ✅ 关键词难度 < 你的域名权重能承受的范围
- ✅ 与产品相关性高
- ✅ 商业价值高（能带来转化）
- ✅ 趋势向上或稳定

**2.5 确定核心关键词**

选择标准：
- 1-3个主关键词（网站首页优化）
- 5-10个次级关键词（主要页面优化）
- 50-100+个长尾词（博客文章优化）

### 输出物
- ✅ 关键词总表（Excel/Google Sheets）
  - 包含：关键词、搜索量、难度、CPC、分类、优先级
- ✅ 核心关键词列表（1-3个）
- ✅ 次级关键词列表（5-10个）
- ✅ 长尾关键词库（50-100+个）
- ✅ 关键词优先级矩阵

## 🎯 **阶段3: 竞品分析**

### 目标
了解竞争对手的SEO策略，找到差异化机会

### 具体工作

**3.1 竞品发现**

**方法1: Google 搜索**
- [ ] 搜索核心关键词，记录前10名
- [ ] 搜索次级关键词，记录重复出现的网站
- [ ] 使用 `related:竞品域名` 找相似网站

**方法2: SEO 工具**
- [ ] Semrush: 输入关键词，查看"竞争对手"
- [ ] Similarweb: 查看"相似网站"
- [ ] Ahrefs: "竞争域名"功能

**方法3: 行业目录**
- [ ] Product Hunt 同类产品
- [ ] AlternativeTo 替代品列表
- [ ] GitHub Awesome 列表
- [ ] Slant 对比网站

**方法4: 社区推荐**
- [ ] Reddit 相关讨论中提到的工具
- [ ] Hacker News 讨论
- [ ] Stack Overflow 推荐

**3.2 竞品基础信息收集**

对每个竞品收集：
- [ ] 域名
- [ ] 域名注册时间（Whois查询）
- [ ] 域名权重（Ahrefs DR / Moz DA）
- [ ] 月访问量（Similarweb / Semrush）
- [ ] 流量来源分布（自然搜索/直接/社交/外链）
- [ ] 主要流量国家/地区
- [ ] 页面总数（site:domain.com）

**工具：**
- Whois.com（域名信息）
- Similarweb（流量数据）
- Semrush（SEO数据）
- Ahrefs（外链数据）

**3.3 竞品 SEO 数据分析**

**On-Page SEO：**
- [ ] TDK信息（Title, Description, Keywords）
  - 工具：SEO Meta in 1 Click, MozBar
- [ ] H1-H6 标题结构
- [ ] URL 结构规律
- [ ] 内部链接策略
- [ ] 页面加载速度
- [ ] 移动端适配情况
- [ ] 结构化数据使用

**关键词策略：**
- [ ] 排名关键词总数（Semrush/Ahrefs）
- [ ] Top 10 排名关键词数量
- [ ] 主要排名关键词列表
- [ ] 关键词分布（信息型/交易型）
- [ ] 长尾词策略

**外链分析：**
- [ ] 总外链数量
- [ ] 引用域数量（Referring Domains）
- [ ] 高质量外链来源（DR>50）
- [ ] 外链类型分布（博客/新闻/目录/社交）
- [ ] 锚文本分布
- [ ] 外链增长趋势

**3.4 竞品内容策略分析**

- [ ] 网站结构（查看sitemap.xml）
- [ ] 主要页面类型（首页/产品页/博客/文档）
- [ ] 博客文章数量和频率
- [ ] 内容类型（教程/对比/案例/新闻）
- [ ] 内容深度（平均字数）
- [ ] 多媒体使用（图片/视频/交互工具）
- [ ] 内容更新频率
- [ ] 内容质量评估

**3.5 竞品用户体验分析**

- [ ] 网站导航结构
- [ ] 用户转化路径
- [ ] CTA（行动号召）设计
- [ ] 核心功能/工具
- [ ] 社交证明（评价/案例/数据）
- [ ] 用户互动功能

**3.6 竞品差异化分析**

找到机会点：
- [ ] 竞品的优势是什么？
- [ ] 竞品的劣势/缺失是什么？
- [ ] 哪些关键词竞品排名弱？
- [ ] 哪些内容类型竞品没有覆盖？
- [ ] 哪些外链渠道竞品没有利用？
- [ ] 用户体验上有哪些改进空间？

### 输出物
- ✅ 竞品列表（5-10个主要竞品）
- ✅ 竞品对比表（Excel）
  - 包含：域名、DR、流量、关键词数、外链数、内容策略等
- ✅ 竞品SEO策略总结文档
- ✅ 差异化机会清单
- ✅ 可借鉴的最佳实践列表

## 📐 **阶段4: SEO策略制定**

### 目标
基于前面的调研，制定具体的SEO优化策略

### 具体工作

**4.1 网站结构规划**

- [ ] 确定网站主要页面类型
  - 首页
  - 产品/服务页
  - 功能页（如在线工具）
  - 文档/教程页
  - 博客
  - 关于/联系页

- [ ] 设计URL结构
  ```
  示例：
  domain.com/
  domain.com/converter/
  domain.com/docs/
  domain.com/blog/
  domain.com/blog/category/
  domain.com/blog/post-title/
  ```

- [ ] 规划内部链接结构
  - 首页链接到主要页面
  - 主要页面之间互相链接
  - 博客文章链接到相关文章和主要页面
  - 建立主题集群（Topic Clusters）

**4.2 关键词分配策略**

为每个页面分配关键词：
- [ ] 首页：1-2个核心关键词
- [ ] 主要页面：各1-2个次级关键词
- [ ] 博客文章：各1个长尾关键词

创建关键词映射表：
| 页面URL | 主关键词 | 次要关键词 | 搜索意图 | 优先级 |
|---------|---------|-----------|---------|--------|
| /       | 核心词1  | 核心词2    | 交易型   | P0     |
| /docs/  | 次级词1  | 长尾词1-3  | 信息型   | P1     |

**4.3 On-Page SEO 标准**

制定统一标准：
- [ ] **Title 标签格式**
  - 格式：主关键词 | 次要关键词 | 品牌名
  - 长度：50-60字符
  
- [ ] **Meta Description 格式**
  - 包含主关键词和CTA
  - 长度：150-160字符
  
- [ ] **H1 标签**
  - 每页唯一
  - 包含主关键词
  
- [ ] **URL 规范**
  - 小写字母
  - 用连字符分隔
  - 包含关键词
  - 简短清晰

- [ ] **图片优化**
  - 文件名包含关键词
  - Alt 文本描述性
  - 压缩优化
  - 使用 WebP 格式

**4.4 技术SEO清单**

- [ ] **网站性能**
  - 页面加载时间 < 3秒
  - Core Web Vitals 达标
  - 图片懒加载
  - 代码压缩

- [ ] **移动端优化**
  - 响应式设计
  - 移动端友好测试通过
  - 触摸元素间距合适

- [ ] **安全性**
  - HTTPS 证书
  - 安全头部配置

- [ ] **可抓取性**
  - Robots.txt 配置
  - XML Sitemap 生成
  - 提交到 Google Search Console
  - 提交到 Bing Webmaster Tools

- [ ] **结构化数据**
  - Schema.org 标记
  - 根据内容类型选择合适的 Schema
    - Article（文章）
    - HowTo（教程）
    - FAQPage（常见问题）
    - Product（产品）
    - SoftwareApplication（软件）
    - BreadcrumbList（面包屑）

- [ ] **其他技术要素**
  - Canonical 标签（避免重复内容）
  - Hreflang 标签（多语言网站）
  - Open Graph 标签（社交分享）
  - Twitter Card 标签

**4.5 内容质量标准**

制定内容创作规范：
- [ ] **内容长度**
  - 首页：500-800字
  - 主要页面：800-1500字
  - 博客文章：1500-3000字
  - 深度指南：3000+字

- [ ] **内容结构**
  - 清晰的H2-H6层级
  - 短段落（2-3句）
  - 使用列表和表格
  - 包含目录（长文章）

- [ ] **内容要素**
  - 包含目标关键词（自然分布）
  - 包含LSI关键词（语义相关词）
  - 包含内部链接（3-5个）
  - 包含外部链接（权威来源）
  - 包含多媒体（图片/视频）
  - 包含CTA

- [ ] **内容质量**
  - 原创性
  - 准确性
  - 实用性
  - 可读性（Flesch Reading Ease > 60）
  - 无语法错误

### 输出物
- ✅ 网站结构图
- ✅ URL结构规范文档
- ✅ 关键词映射表
- ✅ On-Page SEO标准文档
- ✅ 技术SEO检查清单
- ✅ 内容创作规范文档

## 📝 **阶段6: 内容规划**

### 目标
制定详细的内容创作计划

### 具体工作

**6.1 内容类型规划**

根据关键词和用户需求，规划内容类型：

**A. 核心页面内容**
- [ ] 首页
- [ ] 产品/功能页
- [ ] 定价页
- [ ] 关于页
- [ ] 联系页

**B. 文档/教程内容**
- [ ] 快速开始指南
- [ ] 完整文档
- [ ] API 参考
- [ ] 代码示例
- [ ] 常见问题（FAQ）

**C. 博客内容类型**
- [ ] **教程类**（How-to）
  - 针对长尾关键词
  - 解决具体问题
  
- [ ] **对比类**（Comparison）
  - 产品对比
  - 方案对比
  
- [ ] **指南类**（Guide）
  - 完整指南
  - 最佳实践
  
- [ ] **案例研究**（Case Study）
  - 用户故事
  - 成功案例
  
- [ ] **列表类**（Listicle）
  - "10个最佳..."
  - "5个技巧..."
  
- [ ] **新闻/更新**（News）
  - 产品更新
  - 行业动态

**6.2 内容主题挖掘**

基于关键词研究，生成内容主题：

**方法1: 关键词转化为主题**
- 关键词："how to convert json to toon"
- 主题："How to Convert JSON to TOON in 3 Easy Steps"

**方法2: 竞品内容分析**
- 查看竞品博客
- 找到表现好的文章
- 创作更好的版本（10x content）

**方法3: 用户问题挖掘**
- Reddit/Quora 问题
- Stack Overflow 问题
- Google "People Also Ask"
- 客户支持问题

**方法4: 内容差距分析**
- 竞品没有覆盖的主题
- 关键词有搜索量但内容少的主题

**6.3 内容创作计划**

制定内容日历：

**发布频率建议：**
- 新网站（0-3个月）：每周3-5篇
- 成长期（3-6个月）：每周2-3篇
- 稳定期（6个月+）：每周1-2篇

**内容优先级：**
- P0：核心页面（首页、产品页、文档）
- P1：高搜索量关键词的文章
- P2：中等搜索量关键词的文章
- P3：长尾关键词文章
- P4：品牌内容、案例研究

**6.4 内容创作流程**

标准化流程：

## ⚙️ **阶段7: 技术SEO配置**

### 目标
确保网站技术层面符合SEO最佳实践

### 具体工作

**7.1 网站基础设置**

- [ ] **域名配置**
  - 选择合适的域名
  - 配置 HTTPS
  - 设置 www 或非 www 重定向
  - 配置 Canonical URL

- [ ] **服务器配置**
  - 选择快速的主机
  - 配置 CDN
  - 启用 Gzip/Brotli 压缩
  - 配置缓存策略

**7.2 网站性能优化**

- [ ] **Core Web Vitals 优化**
  - LCP (Largest Contentful Paint) < 2.5s
  - FID (First Input Delay) < 100ms
  - CLS (Cumulative Layout Shift) < 0.1

- [ ] **页面速度优化**
  - 图片优化（压缩、WebP、懒加载）
  - 代码压缩（HTML, CSS, JS）
  - 减少HTTP请求
  - 使用浏览器缓存
  - 异步加载JS
  - 关键CSS内联

- [ ] **移动端优化**
  - 响应式设计
  - 移动端友好测试
  - 触摸元素大小合适
  - 避免使用Flash

**工具：**
- Google PageSpeed Insights
- GTmetrix
- WebPageTest
- Chrome DevTools Lighthouse

**7.3 可抓取性配置**

- [ ] **Robots.txt**
  ```
  User-agent: *
  Allow: /
  Disallow: /admin/
  Disallow: /private/
  
  Sitemap: https://yourdomain.com/sitemap.xml
  ```

- [ ] **XML Sitemap**
  - 生成 sitemap.xml
  - 包含所有重要页面
  - 设置优先级和更新频率
  - 提交到 Google Search Console
  - 提交到 Bing Webmaster Tools

- [ ] **内部链接结构**
  - 确保所有页面可从首页3次点击内到达
  - 使用描述性锚文本
  - 避免孤岛页面

**7.4 结构化数据实施**

根据页面类型添加 Schema.org 标记：

- [ ] **全站通用**
  - Organization
  - WebSite
  - BreadcrumbList

- [ ] **内容页面**
  - Article
  - BlogPosting
  - TechArticle

- [ ] **产品/工具页**
  - SoftwareApplication
  - Product

- [ ] **教程页面**
  - HowTo

- [ ] **FAQ页面**
  - FAQPage

**验证工具：**
- Google Rich Results Test
- Schema.org Validator

**7.5 URL 结构规范**

- [ ] 使用语义化URL
- [ ] 使用连字符（-）而非下划线（_）
- [ ] 保持URL简短
- [ ] 使用小写字母
- [ ] 避免URL参数（尽可能）
- [ ] 设置 Canonical 标签

**7.6 重定向管理**

- [ ] 设置 301 重定向（永久）
- [ ] 避免 302 重定向（临时）
- [ ] 避免重定向链
- [ ] 创建自定义 404 页面

**7.7 国际化配置（如适用）**

- [ ] Hreflang 标签
- [ ] 多语言URL结构
- [ ] 地理定位设置

**7.8 安全性配置**

- [ ] HTTPS 证书
- [ ] 安全头部
  - Content-Security-Policy
  - X-Frame-Options
  - X-Content-Type-Options
- [ ] 定期安全扫描

**7.9 监控工具设置**

- [ ] **Google Search Console**
  - 验证网站所有权
  - 提交 Sitemap
  - 监控索引状态
  - 查看搜索表现

- [ ] **Google Analytics 4**
  - 安装跟踪代码
  - 设置目标/转化
  - 配置事件跟踪

- [ ] **其他工具**
  - Bing Webmaster Tools
  - Yandex Webmaster（如适用）
  - Cloudflare Analytics（如使用CDN）

**7.10 技术SEO审计**

定期检查：
- [ ] 爬虫错误（404, 500等）
- [ ] 重复内容
- [ ] 缺失或重复的 Meta 标签
- [ ] 损坏的内部/外部链接
- [ ] 页面加载速度
- [ ] 移动端可用性
- [ ] 结构化数据错误
- [ ] 索引覆盖率

**工具：**
- Screaming Frog SEO Spider
- Ahrefs Site Audit
- Semrush Site Audit
- Google Search Console

### 输出物
- ✅ 技术SEO配置清单
- ✅ 网站性能优化报告
- ✅ 结构化数据实施文档
- ✅ 监控工具设置指南
- ✅ 技术SEO审计报告（定期）
